6.1 Manajemen Proyek & Struktur Direktori

Dalam pengembangan skala industri, struktur direktori bukan sekadar preferensi estetika, melainkan kebutuhan fungsional untuk kolaborasi dan deployment.

6.1.1 Filosofi Penamaan Berkas

Gunakan Lowercase dan Hyphenation (-) untuk konsistensi di server Linux/Unix.

  • Salah: /Foto Profil.JPG
  • Benar: /foto-profil.jpg

6.1.2 Analisis Fungsi Direktori

DirektoriPeran & Fungsi Utama
/srcSource Code: Dapur pengembangan kode mentah.
/distDistribution: Hasil akhir aplikasi siap publish.
/assetsMedia pendukung (images, icons, fonts).
/js/componentsLogika UI berbasis komponen (Modular).

6.2 Standarisasi Kode (Airbnb Style Guide)

Mengadopsi standar de facto industri global untuk menjaga kualitas kode.

// 6.2.1 Manajemen Variabel // GOOD: Gunakan const secara default const API_URL = "https://api.example.com"; // GOOD: Gunakan let hanya jika nilainya berubah let counter = 0; counter++; // 6.2.2 Struktur Data // GOOD: Gunakan sintaks literal const mahasiswa = { nama: 'Alif', nim: '231011', isActive: true, // Trailing comma disarankan }; // 6.2.3 Naming Conventions const namaLengkap = '...'; // camelCase untuk variabel class UserProfile {} // PascalCase untuk Class const MAX_SIZE = 5000; // UPPERCASE_SNAKE_CASE untuk konstanta

6.4 Web Vitals

  • LCP: Loading < 2.5s
  • FID: Respons < 100ms
  • CLS: Stabilitas < 0.1

6.5 Security

Gunakan textContent alih-alih innerHTML untuk mencegah XSS.

6.6 Konfigurasi SEO & PWA

SEO Lanjutan:

Gunakan JSON-LD untuk Schema Markup dan Canonical Tags guna mencegah duplikasi konten.

Komponen PWA:
  • HTTPS (Secure Context)
  • Web App Manifest (JSON)
  • Service Worker (Offline Support)

Gunakan strategi Feature Branch (Git Flow) untuk menjaga kebersihan branch utama.

6.8 Deployment & CI/CD Dasar

Pipeline Stages:

Source → Build → Test → Staging → Production.

Modern Tech:

Docker (Container) & PaaS (Vercel/Netlify).

Strategi Zero Downtime:
  • Blue-Green: Menjalankan dua lingkungan identik secara bergantian.
  • Canary: Rilis ke sebagian kecil user dulu untuk testing.
# .github/workflows/static.yml name: Deploy to Pages on: push: branches: ["main"] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Deploy uses: actions/deploy-pages@v4
