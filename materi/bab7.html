7.1 Implementasi Metodologi Desain Frontend

Penulisan kode tidak dapat dipisahkan dari prinsip UI/UX. Berdasarkan proyek SonicPulse dan NeonVerse, kita menerapkan metodologi terstruktur untuk konsistensi visual.

7.1.1 Atomic Design dalam Tailwind

Mengadopsi prinsip Brad Frost untuk mengelola komponen:

  • Atoms: Kelas utilitas (bg-blue-500, p-4).
  • Molecules: Tombol pencarian (Input + Ikon).
  • Organisms: Navbar responsif (Logo + Nav + Toggle).
  • Templates/Pages: Landing Page utuh (SonicPulse).

7.1.2 Psikologi Visual

Analisis pada proyek SonicPulse:

  • Warna Gelap (Slate-900): Kesan premium, elegan, dan fokus (immersive).
  • Warna Indigo: Kreativitas dan kemewahan modern untuk elemen CTA.
  • Tipografi (Inter/Poppins): Sans-Serif geometris untuk readability tinggi di layar digital.

7.3 Analisis Core Web Vitals

Menganalisis tiga metrik vital dari Google untuk performa website optimal.

LCP (Largest Contentful Paint)

Mengukur kecepatan loading elemen terbesar (Hero Image). Solusi: Gunakan format modern seperti WebP (lebih kecil 30-50%).

CLS (Cumulative Layout Shift)

Mengukur stabilitas visual. Pergeseran dicegah dengan selalu menyertakan atribut width dan height pada gambar.

FID (First Input Delay)

Mengukur responsivitas. Penggunaan Vanilla JS memberikan FID yang unggul karena meminimalkan proses pustaka berat.

7.4 Vanilla JS vs Framework Modern

Meskipun framework populer, memahami fundamental Vanilla JS tetap krusial untuk kontrol penuh aplikasi.

7.4.1 Kelebihan Pendekatan Vanilla JS

  • Performa Tinggi: Tanpa overhead kode pustaka eksternal.
  • Pemahaman Fundamental: Memahami cara kerja DOM secara langsung.
  • Tanpa Build Tools: Cukup file .html dan .js untuk dijalankan.
